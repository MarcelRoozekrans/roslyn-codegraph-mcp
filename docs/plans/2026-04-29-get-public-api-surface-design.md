# `get_public_api_surface` — Design

**Date:** 2026-04-29
**Backlog source:** `docs/BACKLOG.md` § 2 (API & breaking-change tools)

## Goal

Enumerate every `public` and `protected` type and member declared in the production projects of the active solution, returning a flat, deterministically-sorted list suitable for API review or for diffing against a baseline.

This is the foundational step for the larger `find_breaking_changes` tool: once we have a stable, reproducible per-project API surface, "diff today's surface against a stored baseline" is a one-pass set comparison.

## Why

Library authors and anyone reviewing a version bump need a structured answer to "what is the public surface of this library?" Today, agents have to call `get_symbol_context` per known type, or grep for `public class`/`public T` patterns, neither of which gives a complete or canonical view. `Microsoft.CodeAnalysis.PublicApiAnalyzers` covers the same ground but produces a `PublicAPI.txt` file via `dotnet build` — too heavy for a quick agent question.

This tool is also the cheapest item in the `find_breaking_changes` story: enumeration is a Roslyn `INamedTypeSymbol` walk; comparison comes later.

Types:
- `Class`, `Struct`, `Interface`, `Enum`, `Record`, `RecordStruct`, `Delegate`

Members:
- `Constructor`, `Method`, `Property`, `Indexer`, `Field`, `Event`, `Operator` (covers `op_Addition`, `op_Implicit`, etc.)

## What counts as "API surface"

### Per the Standard scope decision

**Included:**
- `public` types declared in source within a non-test, non-generated project
- `public` members declared on public types (declared in source, non-implicit)
- `protected` members declared on **non-sealed** public types (visible to subclasses outside the assembly via inheritance)

**Excluded:**
- `internal` types and members
- `private` members
- Inherited members (per Standard scope — only the members declared on the type itself)
- Compiler-generated members (`IsImplicitlyDeclared`)
- Members in generated source files (`<auto-generated>`, `.g.cs`, `.designer.cs`)
- Members in test projects (per `TestProjectDetector`)
- Sealed-type protected members (a sealed type's protected members can't be reached from outside — treat as internal)

### Edge cases

| Case | Behaviour |
|------|-----------|
| `public sealed class Foo { protected void Bar() {} }` | `Bar` is **excluded** — sealed-type protected can't be reached externally. |
| `public abstract class Foo { protected abstract void Bar(); }` | `Bar` is **included** — subclassable, protected is real API. |
| Record positional properties (auto-synthesized from `public record Foo(int X)`) | **Included** — they ARE public API even though synthesized. Roslyn flags them with `IsImplicitlyDeclared = false` (the syntax declaration counts), so the standard filter works. |
| Record `Equals` / `GetHashCode` / `EqualityContract` (synthesized) | **Excluded** — `IsImplicitlyDeclared = true`. Consumers conceptually know they exist. |
| Static class | Treated as `Class` kind. Static members appear normally. |
| Nested types | Walked recursively — each nested public type produces its own entry. |
| Generic types | The unbound type appears once; the FQN includes the type-parameter list (`MyApp.Container<T>`). Constructed types do not appear separately. |
| Generic methods | Method's name includes type-parameter list (`MyApp.Foo.Bar<TKey>(TKey)`). |
| Operators / conversions | Kind = `Operator`. Name includes the operator (`MyApp.Foo.op_Addition(MyApp.Foo, MyApp.Foo)`). |
| Indexers | Kind = `Indexer`. Name includes parameter types (`MyApp.Foo.this[int]`). |
| Properties (get/set) | Single `Property` entry per property. Accessor visibility is captured in the entry's `accessibility` (most-permissive of the accessors). Asymmetric accessors (e.g., `public get`, `protected set`) — entry shows `Public`; the consumer drills in via `get_symbol_context` if they need accessor-level detail. |
| Conversion operators (`public static implicit operator T(...)`) | Kind = `Operator`. |
| Finalizers | Excluded — private by language definition; not API. |

## Algorithm

Single forward sweep per non-test compilation:

```
for each (projectId, compilation) in loaded.Compilations:
    if testProjectIds.Contains(projectId): skip
    
    walk every type in compilation.Assembly.GlobalNamespace:
        if not (Public type | nested-public type with public-or-protected ancestors): skip
        if not declared-in-source-and-not-generated: skip
        if IsImplicitlyDeclared: skip
        
        emit entry for the type
        
        for each member:
            if accessibility not in {Public, Protected}: skip
            if Protected and containing type is sealed: skip
            if not declared-in-source-and-not-generated: skip
            if IsImplicitlyDeclared: skip
            
            emit entry for the member
```

After collection: sort by `name` ASC ordinal. Build summary buckets.

### Performance

Roughly equivalent to `find_uncovered_symbols`'s candidate-enumeration step (which already walks public types in non-test projects). On a 5-project test fixture, expected sub-millisecond.

## Hardcoded constants

| Constant | Value | Why hardcoded |
|----------|-------|---------------|
| Skip test projects | true | API of test fixtures is not what library authors care about. |
| Skip generated code | true | Auto-generated symbols aren't deliberate API. |
| Skip compiler-generated | true | `IsImplicitlyDeclared`. |
| Sort | `name` ASC ordinal | Deterministic for diffing. |
| Accessibility filter | `Public` and (`Protected` & containing type non-sealed) | Per Standard scope decision. |

No tool parameters, no env vars. Same justification as the rest of this codebase.

### Fixture additions

The existing `TestLib` already has:
- `Greeter` (class) with `Greet`, `OldGreet`, `ComputeLength`, `Dispose`, `GreetFormal`, `FormalNameLength`, `ClassifyName` (public methods/property)
- `IGreeter` (interface)
- `FancyGreeter` (class extending Greeter)

Plus `TestLib2`, `AsyncFixture`, `DisposableFixture` provide variety.

For richer kind coverage we need to add:
- A `record` and a `record struct` (currently no records in the fixture)
- An `enum`
- A `public delegate`
- A `public sealed class with protected member` (negative — protected should be excluded)
- A `public abstract class with protected member` (positive — protected included)
- An indexer (`public T this[int]`)
- A user-defined operator (`public static MyType operator +(MyType a, MyType b)`)

Add these to a small new file `TestLib/PublicApiSamples.cs`. No new project needed — TestLib is fine.

## Out of scope (explicit non-goals)

- **`find_breaking_changes` itself** — the diff-against-baseline tool is a follow-up that consumes this output.
- **Modifiers / return-type fields per entry** — defer until `find_breaking_changes` needs them.
- **Project / namespace filters** — whole-solution; add only on demand.
- **Inherited members shown per type** — Standard scope only declares.
- **PublicAPI.txt format output** — JSON only; the consumer can post-format if needed.
- **`internal` types or members** — out of scope by the "public API surface" definition.
- **Symbol XML doc comments** — interesting but not strictly part of the surface; defer.
- **Documentation comments on accessibility** — defer.

## Decisions log

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Scope | `public` + `protected` (non-sealed) | Matches `Microsoft.CodeAnalysis.PublicApiAnalyzers`; protected IS API |
| Output shape | Summary + flat list | Same convention as `find_async_violations`, `find_uncovered_symbols`; diff-friendly |
| Sort | `name` ASC ordinal | Deterministic; foundational for `find_breaking_changes` |
| Inheritance | Show declared only | Effective surface is computable; repeating bloats output |
| Per-entry detail | Minimal (kind, name, accessibility, project, file, line) | Enough for review and diff identity; richer fields later if needed |
| Sealed-protected | Excluded | Can't be reached from outside |
| Compiler-generated | Excluded | `IsImplicitlyDeclared`; record positional properties pass through |
| Test projects | Excluded | API of test fixtures isn't library API |
| Filters / parameters | None in v1 | Project's convention |
